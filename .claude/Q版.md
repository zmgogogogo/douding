「照片转 Q 版拼豆」功能开发说明文档
文档信息
表格
项目	内容
功能名称	照片转 Q 版拼豆（用户自选风格模式）
方案定位	方案三：用户手动选择风格模板，系统基于预设风格参数，自动完成「AI Q 版化 + 拼豆像素化」全链路生成，输出可直接制作的拼豆图纸
核心优势	效果稳定可控、用户操作门槛低、风格可扩展、无缝复用现有拼豆算法流水线
依赖模块	现有拼豆像素化流水线、AI 图像生成 / 卡通化接口、图片上传模块、画板编辑器
文档版本	V1.0
一、功能概述
1.1 功能目标
解决真实照片直接像素化杂点多、细节碎、不像拼豆的痛点，通过「用户选择风格模板 → AI 按风格生成 Q 版卡通图 → 自动匹配对应拼豆算法参数」的流程，实现真人 / 宠物照片一键生成干净、可爱、可制作的 Q 版拼豆图纸。
1.2 核心设计原则
风格参数化：每种 Q 版风格对应一套完整的「AI 生成参数 + 拼豆算法参数」，用户仅需点选风格，无需手动调节专业参数。
全链路可控：AI 输出严格遵循拼豆友好规范（纯色块、硬描边、少渐变），从源头减少杂色，再进入标准化拼豆流水线。
无缝对接现有系统：生成结果直接输出标准二维颜色矩阵，无缝对接现有画板编辑器，支持后续手动修改。
可扩展架构：风格模板支持配置化新增，无需修改核心代码，可快速迭代新风格。
二、整体业务流程
plaintext
用户上传照片 → 选择Q版风格模板 → 确认生成
    ↓
后端校验图片（人脸/主体检测、清晰度判断）
    ↓
按风格模板拼接Prompt，调用AI卡通化接口生成Q版图
    ↓
AI输出图做前置扁平化预处理（去渐变、强化描边）
    ↓
自动匹配该风格对应的拼豆算法参数，执行像素化流水线
    ↓
生成标准拼豆图纸（二维颜色矩阵 + 色号清单）
    ↓
前端展示结果，支持「重新生成」「进入编辑器修改」「保存图纸」
三、前端交互设计
3.1 页面流程
共 4 个核心页面 / 状态：上传页 → 风格选择页 → 生成加载页 → 结果预览页
3.1.1 照片上传页
核心元素：
上传区域：支持拍照、相册选择，提示「建议上传正面清晰、背景简洁的照片」
裁剪框：自动识别主体，支持手动调整裁剪范围，固定输出比例（默认 1:1 头像，可选 3:4 半身）
基础提示：说明不同风格的适配场景，引导用户选择合适的照片
校验规则：
图片最小分辨率：500×500px，低于则提示清晰度不足
人像风格自动检测人脸，无人脸则提示「未检测到人脸，建议更换照片」
支持 JPG/PNG 格式，大小≤10MB
3.1.2 风格选择页
布局：横向卡片式排列，每个风格一张预览样图 + 风格名称 + 简短描述 + 适用场景标签。

居中展示最终拼豆图纸，支持双指缩放查看细节

五、AI 处理模块设计
5.1 技术选型
初期方案：调用第三方 AI 生图 / 卡通化 API（如通义万相、文心一格、Stable Diffusion API），通过 Prompt 工程控制输出风格，开发快、成本低。
进阶方案：部署微调后的专属卡通化模型，效果更统一、调用成本更低，适合用户量增长后切换。
5.2 处理流程
图片预处理
按用户裁剪区域裁切图片，统一缩放到 AI 接口最优分辨率（如 1024×1024）
人像风格自动人脸居中对齐，提升生成准确度
Prompt 拼接
基础 Prompt = 风格模板正向 Prompt + 参考图控制参数
反向 Prompt = 风格模板反向 Prompt + 通用拼豆避坑词（渐变、噪点、模糊、复杂纹理）
生成调用
采用图生图模式，以上传照片为参考图，按风格参数生成 Q 版图
生成数量：默认生成 1 张，可扩展「多生成几张」选项
输出后处理
自动扣除背景，替换为纯白色
强化轮廓描边，提升边缘对比度
轻微色彩量化预处理，合并相近色，减少后续拼豆杂点
5.3 输出要求
格式：PNG，背景透明 / 纯白
分辨率：≥1024×1024px
硬性要求：无明显渐变、边缘清晰、色块平整，符合拼豆转图输入标准
六、拼豆算法层对接
6.1 对接方式
AI 生成的 Q 版图作为输入，直接传入现有拼豆像素化流水线，全程自动执行，无需用户干预。
流水线执行顺序严格遵循现有规范：
plaintext
保边降噪 → 语义分区（按风格配置决定是否开启） → 分区K-Means量化 → 拼豆色板映射 → 最近邻像素化 → 后处理净化 → 轮廓补强
6.2 参数自动适配
所有算法参数从风格模板的 bean_algorithm_params 中读取，核心适配规则：
降噪强度：贴纸风等高色块风格开启强降噪，简约风格保留细节用弱降噪
颜色数量 K 值：严格按风格配置执行，禁止颜色过多导致色块细碎
分区开关：人像 / 宠物风格开启语义分区，减少跨区串色；简约贴纸风可关闭，提升速度
后处理强度：色块风格提高连通域阈值，保证干净无杂点；细节风格降低阈值，保留细节
强制规则：所有风格全程关闭抖动，统一使用最近邻插值，从根源杜绝杂点
6.3 结果输出
输出标准数据结构，与现有画板完全兼容：
二维颜色矩阵：grid[row][col] = color_id
颜色映射表：色号、RGB 值、豆子用量统计
图纸元信息：尺寸、颜色总数、豆子总数
七、后端服务设计
7.1 架构设计
采用异步任务模式，避免 AI 生成时长导致接口超时。
接口层：接收前端请求，参数校验，返回任务 ID
任务队列：使用消息队列缓存生成任务，削峰填谷
AI 调度层：调用第三方 AI 接口，处理生成结果
算法执行层：调用拼豆算法服务，执行像素化流水线
状态存储：Redis 存储任务状态与中间结果，生成完成后持久化到数据库
7.2 核心接口说明
1. 风格列表查询接口
返回：所有风格模板的配置信息（ID、名称、封面、描述、推荐尺寸）
2. 生成任务提交接口
入参：图片文件 / 图片 URL、风格 ID、自定义尺寸（可选）
返回：任务 ID task_id
3. 任务状态查询接口
入参：task_id
返回：任务状态（pending/processing/success/fail）、进度、失败原因、结果图纸信息（成功时返回）
4. 结果保存接口
入参：task_id、图纸名称
返回：保存后的图纸 ID，可直接跳转编辑器
八、异常与边界处理
表格
异常场景	处理逻辑
图片模糊、无人脸 / 主体	提交时直接拦截，提示用户更换清晰照片
AI 生成失败	自动重试 1 次，仍失败则提示「生成失败，请重新上传」，不扣次数
拼豆算法执行异常	返回兜底结果，提示「生成异常，已为你生成基础版本」
生成结果颜色过少 / 过多	自动修正到合理范围，保证拼豆可制作性
并发量过高	进入队列排队，前端提示「当前排队人数较多，请稍候」
用户重复提交	相同图片 + 相同风格，命中缓存直接返回结果，减少重复调用
九、效果验收标准
9.1 Q 版生成质量
人物 / 宠物特征还原度≥70%，能一眼认出原型
无复杂渐变、无细碎纹理，色块平整
轮廓清晰，有明确描边，无模糊毛边
9.2 拼豆图纸质量
杂点率（＜4 像素色块占比）≤ 2%
轮廓连续完整，无明显断裂
颜色数量符合风格配置，偏差≤2 色
五官 / 关键特征清晰可识别，无崩坏
9.3 性能指标
单张图整体生成耗时 ≤ 15 秒
接口成功率 ≥ 98%
前端操作响应延迟 ≤ 200ms
