一、产品概述
1.1 解决的核心痛点
传统全局像素化算法处理人像、实景照片时，普遍存在三大硬伤，直接导致拼豆成品无法制作：
背景杂乱杂点多：实景背景纹理、杂物多，转像素后生成大量细碎杂色，清理成本极高，主体与背景边界模糊。
肤色斑驳不干净：人脸皮肤纹理、光影渐变、斑点瑕疵会被量化成大量零散杂色，成品脸部分布 “麻子点”，完全失去卡通拼豆的干净质感。
文字模糊难辨认：图片中的文字转像素后边缘毛躁、笔画断裂，无法识别，缺少针对性优化。
跨区域串色：全局统一量化时，背景色、肤色、衣物色互相渗透，比如脸上混入背景色、头发丝飘进肤色区。
1.2 核心设计原则
先分割后量化原则：先通过图像识别技术拆分出背景、人脸、五官、衣物、文字等独立区域，再分区做颜色量化，从根源杜绝跨区串色。
分区差异化处理原则：不同区域执行不同的优化策略 —— 背景强制纯色、肤色平滑统一、文字边缘强化、衣物保留层次，兼顾干净度与层次感。
零抖动硬量化原则：全程关闭误差扩散抖动，所有区域均采用纯色硬量化，从根源杜绝椒盐杂点。
实体可制作原则：最终颜色严格映射到真实拼豆色号，不生成无法采购的中间色，输出即可直接制作。
参数可调节原则：背景色、肤色统一强度、颜色数等关键参数支持用户手动调节，兼顾自动化效果与自定义需求。
1.3 技术能力总览
本功能并非单一 OCR 技术，而是以「语义分割为核心、人脸检测 + OCR 为专项优化」的组合技术方案：

三、图像预处理模块
3.1 目标
统一图片尺寸、去除原图噪声、保留硬边缘，为后续分割与量化提供高质量输入。
3.2 核心算法与参数
1. 尺寸归一化
将输入图长边统一缩放到 1024px，短边按比例自适应，保证分割模型精度的同时控制计算量。
超小图（长边＜500px）先做超分重建，避免细节丢失；超大图（长边＞2000px）先压缩，提升处理速度。
2. 保边降噪
首选算法：引导滤波（Guided Filter）
选型理由：比双边滤波边缘保持能力更强，既能抹除皮肤纹理、背景细碎噪点，又完全不破坏人脸轮廓、五官、文字边缘。
拼豆人像专用参数：
滤波半径 r = 5
正则化系数 eps = 0.025
降级方案：双边滤波 cv2.bilateralFilter(d=5, sigmaColor=40, sigmaSpace=20)，适配低算力端侧场景。
❌ 绝对禁用：高斯模糊、均值模糊 —— 会糊化五官与文字边缘，生成大量过渡杂色。
四、多维度语义识别分割模块（核心技术层）
4.1 背景与主体分割
技术选型
云端方案：PP-HumanSeg v2 轻量人像分割模型，适配人像照片；通用语义分割模型适配宠物、物品等非人像主体。
端侧方案：MODNet 轻量抠图模型，可编译为 WASM 在前端本地运行，无需上传图片。
执行步骤
输入预处理后的图片，模型输出前景概率图（0~255 灰度图，数值越高代表属于主体的概率越大）。
二值化生成掩码：设置阈值 128，大于阈值判定为主体区域，小于阈值判定为背景区域。
掩码形态学优化：
执行 3×3 形态学闭运算，填充主体内部的细小孔洞；
执行 1×1 腐蚀，消除边缘半透明过渡带，保证边缘锐利。
输出标准二值掩码：background_mask，0 为背景，255 为主体。
效果要求
人像发丝、衣物边缘分割精准，无明显缺漏；
背景无残留主体碎片，主体无多余背景孔洞；
边缘无明显锯齿与半透明过渡。
4.2 人脸检测与区域拆分
技术选型
人脸检测：MTCNN / RetinaFace，轻量高精度，支持多人脸检测。
人脸关键点：68 点人脸关键点模型，精准定位眉眼口鼻轮廓。
执行步骤
人脸检测：识别图片中的所有人脸，输出人脸 bounding box，生成人脸整体区域掩码。
关键点定位：定位 68 个关键点，拆分出眼睛、眉毛、嘴巴、鼻子区域，生成「五官掩码」。
皮肤区域提取：
在人脸整体区域内，扣除五官掩码；
叠加 HSV 肤色阈值二次校验（H:5~25, S:20~180, V:120~255），精准剔除胡须、痣、阴影等非皮肤区域；
生成最终的 face_skin_mask 人脸皮肤掩码。
区域互斥校验：皮肤掩码、五官掩码、背景掩码两两互斥，单个像素只能归属一个区域，避免后续量化串色。
多人脸处理
自动检测所有人脸，统一执行肤色优化；
支持用户手动框选指定人脸，单独调整肤色参数。
4.3 OCR 文字检测与增强
技术选型
文字检测：PaddleOCR / DBNet，精准定位图片中的文字位置与轮廓。
可选文字识别：识别文字内容，用于后续像素字体替换优化。
执行步骤
文字检测：输入图片，输出所有文字区域的四边形 bounding box，生成 text_mask 文字掩码。
文字区域标记：将文字区域从主体区域中单独拆分，后续量化时单独处理，不与背景 / 肤色合并。
文字边缘强化：对文字区域做对比度拉升、边缘锐化，保证后续像素化后笔画清晰。
可选优化：识别文字内容后，用标准像素字体重新生成对应文字，完全替代原图文字，效果最优（适合文字转拼豆专属场景）。
五、分区独立优化模块（核心效果层）
5.1 背景区：统一纯色处理
优化目标
彻底消除背景纹理、杂物、渐变，输出单一纯色背景，从根源消除背景杂点。
处理逻辑
默认自动统一：
提取原图背景区域的主色（占比最高的颜色）作为基准色；
将背景掩码内的所有像素，统一替换为基准纯色；
与主体交界处做 1 像素硬化处理，消除半透明过渡灰边。
用户自定义背景色：
提供取色器与预设色卡，用户可指定任意颜色作为背景；
支持透明背景选项，导出 PNG 时背景透明。
边缘优化：
主体边缘做 1 像素的收缩修正，避免背景色渗入主体边缘；
消除抠图常见的 “白边”“灰边” 问题，保证主体轮廓干净利落。
参数可调节
背景模式：自动取色 / 自定义纯色 / 透明背景 / 保留原背景
边缘硬度：0~100，数值越高边缘越锐利，越低越柔和
5.2 人脸皮肤区：肤色统一平滑
优化目标
消除皮肤纹理、斑点、光影渐变、暗沉不均，将复杂的人脸肤色简化为 2~4 层均匀色块，既保证干净无杂点，又保留基础明暗层次。
核心处理算法
第一步：肤色保边平滑
算法：引导滤波，以皮肤掩码为约束，仅在皮肤区域内做平滑处理。
作用：抹除雀斑、痘印、毛孔、皮肤纹理等细碎细节，同时严格保护五官边缘，不糊化眼睛嘴巴。
强度参数：
轻度：滤波半径 3，eps=0.015，保留轻微皮肤层次
中度（默认）：滤波半径 5，eps=0.025，大部分纹理消失，肤色均匀
重度：滤波半径 8，eps=0.04，完全磨平，极致干净，适合 Q 版卡通风格
第二步：肤色分层量化（皮肤区专属）
在皮肤区域内单独执行 K-Means 聚类，强制将肤色压缩为固定层数：
极简模式：2 色（主肤色 + 阴影色），最干净，适合小尺寸拼豆
标准模式：3 色（高光 + 主肤色 + 阴影色），兼顾干净与立体感
精细模式：4 色（高光 + 主肤色 + 暗部 + 腮红），适合大尺寸人像
全程无抖动硬分配，无任何过渡杂色。
第三步：肤色校准统一
自动白平衡矫正，修正偏黄、偏蓝、偏红的肤色；
统一肤色饱和度与亮度，避免局部暗沉、局部过曝；
支持用户手动调节肤色冷暖、深浅。
五官保护机制
五官区域（眼睛、眉毛、嘴巴）单独掩码隔离，不参与肤色平滑与聚类；
五官边缘做硬化处理，保证轮廓清晰，避免被肤色糊化。
5.3 五官 / 文字区：细节强化
对眼睛、嘴巴、文字区域做边缘锐化，提升对比度；
量化时保留更多颜色层数，保证细节辨识度；
后处理时单独做轮廓补强，避免像素化后笔画断裂、五官变形。
5.4 其他主体区（衣物、头发等）
执行保边降噪，去除细碎纹理；
按区域复杂度设置对应 K 值，大面积色块 K 值小，细节丰富区域 K 值略高；
头发区域单独优化，减少发丝细碎杂色，合并为大块发色。
六、分区颜色量化模块
6.1 核心规则
所有区域独立执行 K-Means 聚类，颜色互不干扰，彻底解决跨区域串色问题。全程强制关闭所有抖动 / 仿色，使用纯色硬分配。
6.2 各区域 K 值配置（默认值，支持用户调节）
表格
区域	默认 K 值	说明
背景区	1	强制纯色，不允许多色
人脸皮肤区	3	高光 + 主色 + 阴影，可切换 2/3/4 层
五官区	4~6	眼睛、眉毛、嘴巴分别保留独立颜色
头发区	3~4	主发色 + 高光 + 暗部
衣物 / 其他主体	4~6	按图案复杂度自适应
文字区	2	文字色 + 背景色，保证清晰
6.3 算法实现
按各区域掩码提取对应像素；
每个区域单独执行 MiniBatchKMeans 聚类，提升计算速度；
每个像素硬分配到距离最近的聚类中心，无误差扩散；
按掩码回填各区域，得到全局量化后的纯色图。
6.4 色板映射
将所有聚类中心颜色转换到 LAB 色彩空间；
使用 CIEDE2000 色差公式 与官方拼豆色板逐一匹配，取色差最小的色号；
色差阈值：ΔE＜5 为合格匹配，ΔE＞10 时提示用户该颜色无对应色号，可选择近似替代或保留自定义色；
全图像素替换为最终的拼豆色板颜色。
七、像素化与后处理模块
7.1 边缘对齐最近邻像素化
核心算法：边缘对齐最近邻插值
基础缩放：使用最近邻插值 cv2.INTER_NEAREST，是唯一不生成过渡色的缩放算法，将量化图缩放到用户指定的拼豆尺寸。
边缘对齐优化：
提前提取主体轮廓、五官轮廓、文字轮廓；
微调缩放原点，让轮廓线尽可能落在像素格的边界上；
避免轮廓横穿像素格导致的半格混色、边缘毛躁。
尺寸规则：
用户可指定宽度 / 高度像素数，或选择预设尺寸（小 / 中 / 大）；
保持原图比例，另一维度自动取整。
❌ 绝对禁用：双线性、双三次、Lanczos 插值 —— 会生成大量过渡中间色，量化后全部变为杂点。
7.2 后处理杂点净化
算法 1：带区域约束的连通域面积过滤
原理：统计每个同色连通域的像素面积，小于阈值的微小色块判定为噪点，替换为周边占比最高的主色。
分区差异化阈值：
背景区：阈值 2 像素，任何小杂点全部清除
皮肤区：阈值 3 像素，保证面部干净
五官 / 文字区：阈值 1 像素，仅清除孤立椒盐点，保留细节
实现步骤：
对每种颜色分别执行连通域分析 cv2.connectedComponentsWithStats；
筛选面积＜阈值的连通域；
对每个杂色块，取其 3×3 邻域内占比最高的主色填充。
算法 2：形态学优化
背景 / 皮肤区：执行 2×2 形态学开运算，清除 1 像素边缘毛刺；
五官 / 文字区：不做形态学操作，避免侵蚀细节。
7.3 轮廓与细节补强
主体轮廓补强：检测外轮廓断裂点，距离≤3 像素的断点自动连接补全，保证轮廓连续干净。
五官细节修复：
检测眼睛、嘴巴位置，修正像素化后的变形；
保证左右眼对称、嘴巴轮廓完整，提升人像辨识度。
文字笔画修复：补全断裂的笔画，统一笔画宽度，保证文字清晰可辨。

智能优化参数（核心）
背景统一
开关：开启 / 关闭背景处理
背景模式：自动取色 / 自定义纯色 / 透明 / 保留原背景

9.2 部署方案
云端方案：模型部署在服务端，前端上传图片，后端处理后返回结果，效果最好，功能最全。
端侧方案：轻量模型编译为 WASM，前端本地执行，无需上传图片，响应快、隐私性好，适合简单场景。
混合方案：基础转图本地执行，智能优化（分割、人脸、OCR）云端执行，兼顾体验与效果。

十一、异常与边界处理
表格
异常场景	处理逻辑
未检测到人脸	自动关闭肤色优化，提示「未检测到人脸，已跳过肤色统一」，正常执行背景统一与转图

模型推理失败	降级为标准全局转图模式，提示「智能优化暂时不可用，已为你生成基础版本」
肤色匹配色号色差过大	提示用户该肤色无完美匹配色号，推荐近似色，支持手动替换
