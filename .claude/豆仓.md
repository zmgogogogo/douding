一、产品概述
1.1 模块定义
豆仓是用户的「线上虚拟拼豆仓库」，1:1 对应用户手中的实体拼豆库存。用户可录入、管理自己拥有的所有拼豆颜色、规格、数量，系统自动将图纸与豆仓库存做智能匹配，给出缺色提醒、替代色推荐、补购建议，并自动记录制作消耗，实现拼豆全生命周期的数字化管理。

1.3 核心设计原则
色号标准化原则：内置主流品牌官方色号库，颜色参数精准校准，避免用户手动输入颜色偏差，保证匹配准确度。
实体对齐原则：所有库存数据严格对应用户实体豆子，支持多规格、多品牌、多仓库独立管理，贴近真实使用场景。
全链路打通原则：与转图、编辑、导出、制作全流程打通，图纸生成即匹配库存，制作完成自动扣减，无需用户手动计算。
智能辅助原则：基于色差算法提供替代色推荐、智能补购建议、消耗预测，降低用户决策成本。
高效录入原则：提供整包快速添加、批量导入、扫码录入等多种录入方式，最大程度降低用户库存录入门槛。

三、核心功能模块详细设计
3.1 基础色板库模块（系统底层支撑）
豆仓的基础数据层，由平台官方维护，所有用户豆仓的色号均来源于此，保证颜色标准统一、匹配精准。
3.1.1 品牌与色号体系
内置主流品牌
国内品牌：MARD（拼豆）、咪小窝、元派
海外品牌：Perler、Hama、Artkal、Seya
特殊品类：夜光、荧光、透明、金属、磨砂等特殊效果色
色号唯一标识
系统内部色号 ID：全局唯一，跨品牌统一管理
品牌原生色号：各品牌官方色号（如 M01、P45）
同色映射：不同品牌的相同 / 近似颜色建立映射关系，支持跨品牌匹配
每个色号的完整属性
表格
属性字段	类型	说明
color_id	bigint	系统全局唯一色号 ID
brand	varchar	所属品牌
original_code	varchar	品牌原生色号
color_name	varchar	颜色官方名称
rgb	array	[R,G,B] 标准色值

is_discontinued	boolean	是否停产

3.1.2 包装规格库
内置官方标准包装规格，支持用户一键整包录入，无需手动输数量。
规格维度：
豆子直径：5mm（标准）、2.6mm（迷你）、3mm、8mm（超大）
包装规格：100 颗装、500 颗装、1000 颗装、整包 500g、整包 1kg
换算系数：
系统内置标准换算：5mm 拼豆 ≈ 10 颗 / 克；2.6mm 拼豆 ≈ 50 颗 / 克
支持用户自定义换算系数，适配不同品牌豆子重量差异
录入支持：选择品牌 + 色号 + 规格，自动带出默认数量，支持手动修改
3.1.3 色号维护机制
平台定期更新品牌新色、下架停产色
支持用户提交色号补充申请，管理员审核后加入官方库
颜色参数定期校准，保证 RGB/LAB 值与实体豆子一致

用户个人库存的主管理页面，是豆仓系统的核心操作入口。

页面布局：顶部数据概览栏 + 筛选搜索栏 + 色号卡片网格 + 底部操作栏

筛选与搜索
搜索框：支持按色号、颜色名称、品牌原生号模糊搜索
品牌筛选：下拉选择单一品牌 / 全部品牌
规格筛选：5mm/2.6mm / 全部规格
色系筛选：按红 / 橙 / 黄 / 绿 / 青 / 蓝 / 紫 / 灰 / 棕 / 黑白快速筛选
类型筛选：基础色 / 荧光色 / 透明色 / 金属色等
排序方式：按数量升序 / 降序、按色号排序、按色系排序、按使用频率排序

3.2.2 库存录入方式
提供 6 种录入方式，覆盖不同用户场景：
整包快速添加
流程：选择品牌 → 选择规格 → 勾选色号 → 确认添加
支持多选色号批量添加，默认数量为整包标准数量，可手动调整
热门色号前置推荐，提升选色效率
手动单条添加
手动选择色号，输入数量、规格、购入价格、备注
支持选择「按颗录入」或「按克录入」，自动换算
扫码录入（进阶）
扫描拼豆包装上的条形码 / 二维码，自动识别品牌、色号、规格
支持手动修正数量，扫码后一键入库
取色找色录入
用户上传豆子照片 / 取色，系统自动匹配最接近的官方色号
显示匹配度，用户确认后录入数量
Excel 批量导入
下载官方模板，按格式填写色号、数量
上传后自动校验，错误行标红提示，正确行一键导入
支持按品牌色号、系统色号两种导入方式
购物车一键入库
对接商城模块，用户收到货后，订单一键同步入库
自动匹配色号、规格、数量，无需手动录入
3.2.3 库存调整操作
数量增减
点击色号卡片的 +/- 按钮快速调整，步进值可设置（默认 10 颗）
进入详情页精确输入数量修改
批量调整
批量选中色号，统一设置数量 / 增减数量
批量修改品牌、规格、备注
删除色号
单个删除 / 批量删除，删除前二次确认
删除后库存变更记录保留，可在日志中查看
库存盘点
生成盘点表，用户核对实际库存，输入实盘数量
自动计算盘盈盘亏，生成盘点差异报告
确认后批量调整库存，生成盘点记录
3.2.4 多规格与多品牌管理
规格隔离：5mm 与 2.6mm 豆子库存独立计算，不可混用；匹配图纸时按对应规格匹配
品牌切换：用户可设置默认常用品牌，匹配图纸时优先匹配默认品牌
同色合并显示：同一颜色不同品牌可选择合并 / 分开显示，方便用户管理近似色
规格换算：支持查看不同规格的等效数量，辅助采购决策
3.2.5 色号详情页
点击色号卡片进入，展示单款颜色的全量信息：
颜色展示区：大尺寸纯色块，显示色号、名称、品牌、规格
库存信息：当前库存数量、安全阈值、预估可用时长
出入库记录：时间轴形式展示所有库存变更记录，包含时间、变动数量、变动原因（入库 / 制作扣减 / 盘点调整 / 手动修改）
相关图纸：使用了该颜色的历史图纸，按使用量排序
替代色推荐：豆仓内已有的近似颜色，按色差从小到大排序，显示色差值
快捷操作：调整数量、加入心愿单、分享色号、删除
3.3 图纸智能匹配模块（核心业务价值）
打通豆仓与图纸体系，实现库存自动校验、缺色智能提醒，是豆仓模块的核心价值输出点。
3.3.1 单图纸缺色匹配
触发场景：打开任意图纸、转图生成完成、编辑器修改图纸后自动触发
匹配逻辑
提取图纸中所有颜色及其用量（豆子颗数）
与用户当前豆仓库存一一比对
计算每个颜色的库存充足情况
结果展示（图纸页底部豆仓栏）
整体匹配度：豆仓匹配度 82%，即库存充足的颜色占总颜色数的比例
充足颜色：X 种，库存完全满足
不足颜色：X 种，有库存但数量不够
缺失颜色：X 种，豆仓内完全没有
可制作数量：当前库存可制作该图纸的份数
缺色详情弹窗
列表展示所有颜色：色块 + 色号 + 名称 + 图纸用量 + 当前库存 + 缺口数量
不足颜色橙色标注，缺失颜色红色标注
每个缺色旁显示「替代色」按钮，点击展示豆仓内可用近似替代色
替代色推荐
基于 CIEDE2000 色差算法，从豆仓已有颜色中匹配最接近的颜色
显示色差值：ΔE＜5 肉眼几乎无差别，推荐替代；5＜ΔE＜10 轻微色差，可替代；ΔE＞10 色差明显，不推荐
用户可一键替换，图纸自动更换为替代色
豆仓限定转图
开关选项：「仅使用我豆仓内的颜色生成图纸」
开启后，转图算法强制将所有颜色映射到用户豆仓已有色号，不生成新颜色
自动选择色差最小的色号匹配，保证生成的图纸用户无需补豆即可制作
3.3.2 多图纸叠加算量
场景：用户计划一次性制作多张图纸，统一计算总用量与缺色
操作流程：在豆仓中选择多张图纸 → 系统自动叠加所有颜色用量 → 生成总用料清单
输出结果
总豆子数、总颜色数
合并后的缺色清单，按缺口数量排序
总采购预估
适用场景：批量制作、囤货采购前统一核算
3.3.3 制作完成自动扣减
场景：用户完成实体制作后，同步扣减豆仓库存
操作入口：图纸详情页「标记为已制作」按钮
扣减规则
默认按图纸标准用量扣减
支持手动调整实际用量、设置损耗率（默认 5%）
可选择制作份数，批量扣减
扣减校验
库存不足时提示：「当前库存不足，是否继续扣减（将出现负库存）」
可选择部分扣减，或取消操作
扣减后：生成库存变更记录，关联对应图纸，更新库存预警状态
3.3.4 编辑器实时匹配
在图纸编辑器中修改时，实时同步豆仓匹配状态：
顶部状态栏实时显示：当前颜色数 / 豆仓已有颜色数 / 缺色数
新增颜色时，若豆仓没有则高亮提示
提供「一键替换为豆仓颜色」功能，批量替换所有超出色库的颜色
3.4 采购与补给模块
基于库存与缺色数据，提供智能采购建议，打通从缺色到补购的闭环。
3.4.1 缺色清单
自动汇总：自动汇总所有打开过的图纸的缺色，合并去重，计算总缺口
清单内容：色号 + 色块 + 名称 + 缺口总量 + 参考单价 + 小计金额
排序方式：按缺口数量、按色系、按品牌、按价格
批量操作
一键全选加入采购清单
勾选部分颜色加入心愿单
标记为「已有替代色，不采购」
导出采购清单 Excel
3.4.2 智能采购建议
基于历史消耗、库存阈值、常用色推荐，生成采购建议清单：
补货逻辑
低于安全阈值的颜色：推荐补货到阈值数量
高频使用的热门色：推荐适量囤货
图纸缺色：优先推荐
采购量计算
建议采购量 = 安全库存 - 当前库存 + 未来 30 天预估消耗量
安全库存阈值可自定义，默认 200 颗 / 色
推荐包装：自动匹配最接近的整包规格，避免拆分购买
3.4.3 心愿单
用户可将暂时不买、计划采购的颜色加入心愿单
支持备注、优先级设置
降价提醒：对接商城的话，心愿单内颜色降价时推送通知
一键批量结算：心愿单内商品一键加入购物车
3.4.4 采购记录
采购入库记录：每一次采购的时间、色号、数量、单价、总金额
采购统计：月度 / 年度采购总金额、采购颜色数、采购品牌分布
订单同步：对接商城模块，自动同步线上订单采购记录
手动录入：支持手动添加线下采购记录，完善库存流水
3.5 消耗与统计分析模块
多维度统计豆子使用情况，帮助用户掌握消耗规律，合理规划采购。
3.5.1 出入库流水
完整记录所有库存变动：入库、扣减、盘点调整、手动修改
每条记录包含：时间、色号、变动数量、变动类型、关联单据 / 图纸
支持按时间、类型、色号筛选搜索
支持导出流水 Excel
3.5.2 消耗统计
时间维度：日 / 周 / 月 / 年度消耗统计，折线图展示消耗趋势
颜色维度：颜色使用排行榜，柱状图展示 Top20 常用色
色系维度：各色系消耗占比，饼图展示
图纸维度：每张图纸的实际消耗量、损耗率统计
成本维度：周期内消耗的豆子成本，辅助成品定价
3.5.3 库存预警
预警规则
低于安全阈值：黄色预警，提醒补货
库存为 0：红色预警，标记为缺货
负库存：异常预警，提示盘点修正
预警通知
豆仓首页红点提示预警数量
微信 / 站内信推送补货提醒
可自定义预警阈值、是否开启通知
3.5.4 成本核算
库存总价值：按移动加权平均法计算库存总成本
单次制作成本：单张图纸的豆子成本
月度消耗成本：周期内消耗的豆子总价值
支持自定义单价，适配不同采购渠道的价格差异
3.6 高级扩展功能

四、核心算法与计算规则
4.1 色号精准匹配算法
4.1.1 色彩空间选择
采用 CIEDE2000 色差公式 作为颜色匹配的核心算法，相比 RGB 欧氏距离、LAB 欧氏距离，更符合人眼视觉感知，浅色、肤色匹配准确率提升 30% 以上。
所有官方色号提前预计算 LAB 值并缓存，无需实时转换
匹配时直接计算图纸颜色与色板色号的 ΔE00 值
4.1.2 匹配分级规则
表格
色差范围	匹配等级	业务逻辑
ΔE ＜ 3	完全匹配	判定为同色，可直接使用
3 ≤ ΔE ＜ 5	高度近似	肉眼几乎无差别，推荐作为替代色
5 ≤ ΔE ＜ 10	中度近似	存在可察觉色差，可作为次选替代
ΔE ≥ 10	色差明显	不推荐替代，归为缺色
4.1.3 匹配优先级
优先匹配用户设置的默认品牌色号
其次匹配用户豆仓内已有的其他品牌色号
最后匹配全色板库的所有色号
4.2 缺量与可制作份数计算
4.2.1 单颜色缺口计算
plaintext
缺口数量 = 图纸用量 - 当前库存
若缺口 ≤ 0 → 库存充足
若缺口 ＞ 0 → 库存不足，缺口为正数值
若库存 = 0 → 完全缺失

4.2.2 可制作份数计算
plaintext
可制作份数 = floor( min( 各颜色库存数 / 各颜色单份用量 ) )

取所有颜色中可制作份数最少的值，作为整张图的可制作份数。
4.2.3 多图纸叠加用量
相同色号的用量直接累加，合并后统一计算总缺口：
plaintext
总用量 = 图纸A用量 + 图纸B用量 + ...
总缺口 = 总用量 - 当前库存

4.3 库存扣减与成本核算
4.3.1 库存扣减规则
扣减顺序：先进先出（FIFO），优先扣减最早入库的批次
损耗计算：实际扣减 = 图纸用量 × (1 + 损耗率)，损耗率默认 5%，用户可自定义
负库存处理：允许负库存，标记为异常，提醒用户盘点修正
4.3.2 移动加权平均成本
每次采购入库后，重新计算单位成本：
plaintext
新单位成本 = (原有库存总金额 + 本次采购总金额) / (原有库存数量 + 本次采购数量)
每次扣减成本 = 扣减数量 × 当前单位成本

4.4 智能采购推荐算法
4.4.1 安全库存计算
plaintext
安全库存 = 日均消耗量 × 补货周期天数 + 安全冗余量
- 日均消耗量：近30天平均每日消耗数量
- 补货周期：默认7天，用户可自定义
- 安全冗余：默认30颗，应对突发消耗

4.4.2 建议采购量
plaintext
建议采购量 = 安全库存 - 当前库存 + 待制作图纸总缺口
向上取整到最近的整包规格

4.4.3 热门色推荐
基于全平台用户使用数据，结合用户自身色系偏好，推荐用户没有的高使用率热门色，扩充用户色库。
4.5 性能优化算法
色号预缓存：所有官方色号的 LAB 值提前计算缓存，匹配时直接读取，无需实时转换
K 维树加速：色号 LAB 值构建 K-D 树，最近邻搜索速度从 O (n) 提升到 O (logn)，大色板库匹配毫秒级响应
用户库存缓存：用户豆仓颜色列表缓存到 Redis，缺色计算无需查库
增量计算：编辑器修改图纸时，仅计算变更的颜色，不重新全量匹配

十、异常与边界处理
10.1 库存异常
负库存：允许产生，但标记异常，首页红点提示，引导用户盘点修正
零库存：正常显示，红色标注，计入缺货预警
超大量数值：单颜色数量上限设为 99999 颗，防止溢出
重复添加：同色同规格已存在时，自动累加数量，不重复创建记录
10.2 匹配异常
无匹配色号：颜色超出所有色板范围时，标记为「未匹配色」，提示用户可自定义添加
色差过大：所有匹配结果 ΔE 都＞10 时，提示无合适替代色，建议采购
空豆仓匹配：用户豆仓为空时，引导用户添加豆子，不展示匹配结果
10.3 数据一致性
扣减原子性：库存扣减使用数据库事务，保证多颜色扣减要么全成功要么全失败
多端同步：库存变更通过消息队列同步，保证多端数据最终一致
并发扣减：使用乐观锁防止并发修改导致库存不准确
操作留痕：所有库存变更全量记录日志，可追溯可回滚
10.4 导入导出异常
Excel 格式错误：提示具体错误行号与错误原因
色号不存在：标红提示不存在的色号，支持用户提交补充
导入失败回滚：批量导入失败时，不写入任何数据，避免脏数据
