> 调研日期：2026-07-16
> 项目技术栈：Vue 3 + Express + SQLite + Sharp + Tesseract.js

## 一、竞品调研：市面上集成 AI 的拼豆/像素画工具

### 1.1 ohmybead.cn（我嘞个豆）—— 直接竞品

我嘞个豆是目前国内最成熟的拼豆工具，App + Web 双端，**所有功能永久免费**，用户量可观。

**AI 功能清单：**

| 功能             | 描述                                                                         | 实现方式推测                                                   |
| ---------------- | ---------------------------------------------------------------------------- | -------------------------------------------------------------- |
| **图片转图纸**   | 上传任意图片 → AI 自动分析色彩 → 生成像素图纸 → 匹配 Artkal/Perler/Hama 色号 | 后端图像处理 + 色彩量化 + 色卡匹配（类似我们的 image-to-grid） |
| **图例识别**     | 三步流程：网格对齐 → 范围裁剪 → AI 识别色号标注。支持已有拼豆图纸的自动导入  | 计算机视觉网格检测 + OCR 色号识别 + 颜色匹配                   |
| **魔法工具**     | 一键识别最合适的图纸宽度                                                     | 图像分辨率自适应算法                                           |
| **智能库存管理** | 扫码/手动录入豆子库存，追溯消耗来源，自动统计所需豆子                        | 数据库 + 条形码识别                                            |
| **专注模式**     | 高亮单色、标注连续豆数、进度标记、局部放大、屏幕常亮                         | 前端交互增强，非 AI                                            |
| **自动豆数统计** | 自动统计图纸所需颜色和数量，生成采购清单                                     | 纯算法统计                                                     |

**关键洞察：**

- 图例识别是其**差异化核心功能**，v1.6.6 后速度大幅提升，网页端也能使用
- 开发者目标是色号识别准确率达到 99%
- 用户反馈：小图识别较准，部分清晰图可能出现色号偏差，需要手动微调
- 支持中途退出自动保存进度

### 1.2 国际像素画/拼豆工具

#### BeadStudio (Android, datscharf)

- **功能**：照片转拼豆图案，支持 Perler/Hama/Artkal/Nabbi/Ikea
- **实现**：传统图像处理（裁剪 + 对比度/亮度/Gamma 调整 + 抖动 + 颜色匹配），**非 AI 驱动**
- **价格**：$2.49（付费版），免费版含广告
- **板面大小**：1x1 到 522x522 珠子

#### Dreamina (CapCut/剪映)

- **功能**：图片转像素画 + 文字生成像素画
- **实现**：Seedream 4.0 AI 模型，分析颜色/边缘/光照后重建为像素块
- **特点**：支持交互式编辑、局部细化

#### Wplace Pixel

- **功能**：专业照片转像素画平台
- **实现**：严格网格约束 + 调色板限制 + 可选抖动 + SSR 服务端渲染
- **特点**：确定性输出（相同输入总是相同结果）、批量自动化（Wplace Script）

#### Image2pixel

- **功能**：图片转像素画 + 文字生成（Google Gemini API）
- **实现**：可调像素大小/颜色数、支持自定义调色板（.act 格式）、GIF 转换
- **特点**：免费 Web 版 + Steam 桌面版

#### Retro Diffusion (Replicate)

- **功能**：文字生成像素画（4 个专用模型）
- **模型**：
  - `rd-fast`：15 种风格快速生成（肖像到 Minecraft 物品）
  - `rd-plus`：高质量场景/地图/UI/图标
  - `rd-tile`：瓦片集生成（支持变体）
  - `rd-animation`：动画精灵和精灵表
- **特点**：支持输入参考图、调色板图、背景移除、无缝平铺

#### SD-πXL (ETH Zurich, SIGGRAPH Asia 2024)

- **功能**：文字/图片 → 低分辨率量化图像（像素画）生成
- **实现**：Score Distillation Sampling (SDS) + Gumbel-Softmax + 预训练 SDXL 扩散模型
- **特点**：
  - 严格遵循用户指定的调色板
  - 支持 ControlNet（Canny 边缘 + 深度图条件控制）
  - 可生成十字绣、拼豆、马赛克等多种工艺图案
  - 需要 RTX 4090（24GB VRAM），每次生成 1.5-2.5 小时
  - **支持 LoRA 微调**（如 nerijs/pixel-art-xl）

## 二、用户需求与价值分析

### 2.1 AI 创意工具市场趋势

根据 2025 年华泰证券 AI 图像行业报告：

- **AI 原生工具是增长最快的细分市场**：Runway (+150%)、Leonardo.Ai (+200%)、Midjourney (+50%)
- **背景移除**是 AI 图像工具中**使用量最大的功能**（remove.bg 月活 4200 万）
- **文字生成图像**是用户付费意愿最强的功能（Midjourney ARPU $37.5/年）
- AI 将潜在用户从 ~6800 万专业设计师扩展到 40 亿+普通消费者

### 2.2 拼豆用户核心需求排序

基于竞品分析和小红书/应用商店用户反馈：

1. **图片转图纸**（最高频） — 用户想把自己的照片/喜欢的图片做成拼豆
2. **图纸识别导入**（高价值） — 用户在网上找到图纸图片，想快速导入编辑
3. **智能配色建议**（中高频） — 用户不确定用什么珠子颜色更好
4. **文字生成图案**（新兴需求） — 用户用文字描述想要什么图案
5. **风格转换**（小众但高价值） — 用户想把已有设计转成不同风格

## 三、豆丁现有能力盘点

在提出 AI 方案前，先梳理项目已有的技术基础：

| 能力             | 实现                                                         | 状态                           |
| ---------------- | ------------------------------------------------------------ | ------------------------------ |
| 图片转网格       | Sharp 缩放 + CIEDE2000/Oklab 颜色匹配 + Floyd-Steinberg 抖动 | **已实现**                     |
| OCR 图纸识别     | Tesseract.js 色号检测 + 网格检测                             | **已实现**（准确率有提升空间） |
| 网页爬虫         | 小红书链接解析 → 提取图片                                    | **已实现**                     |
| 跨品牌色号转换   | CIEDE2000 跨品牌色差矩阵                                     | **已实现**                     |
| 三层 Canvas 画布 | 参考图层 + 珠子层 + 网格层                                   | **已实现**                     |

- 图片转图纸缺少 AI 主体检测和智能裁剪

- 配色全凭用户手动选择，无智能辅助

## 四、AI 功能方案设计（5 个可落地提案）

### 方案一：AI 智能图片转图纸升级（优先级：P0）

**功能描述：**
在现有图片上传转图纸基础上，增加 AI 智能预处理：

1. **AI 自动去背景** — 上传照片后自动识别主体，移除背景，只保留主角
2. **智能裁剪建议** — AI 分析图片构图，自动推荐最佳裁剪区域
3. **自适应参数推荐** — 根据图片复杂度自动建议网格尺寸和颜色数量
4. **实时预览对比** — 并排展示原图和转换结果，支持参数微调滑块

**技术实现：**

```
前端（Vue 3）：
  ├── ImageImportView.vue 增强
  │   ├── 上传前预处理选项（去背景/不去背景/手动裁剪）
  │   ├── 参数滑块实时调用后端预览 API
  │   └── 原图 ↔ 预览 对比视图
  │
  └── 可选：@imgly/background-removal (WASM 浏览器端去背景)
      └── 优点：隐私保护，无需上传到服务器
      └── 缺点：首次加载 10-20MB 模型，需 WebGPU 加速

后端（Express）：
  ├── POST /api/image/analyze          ← 新增：图片分析接口
  │   └── 返回建议参数（网格尺寸、颜色数、最佳裁剪区域）
  ├── POST /api/image/remove-bg        ← 新增：去背景接口
  │   └── 方案A：sharp + 边缘检测（简单场景）
  │   └── 方案B：@imgly/background-removal-node（复杂场景）
  └── POST /api/image-to-grid（已有）  ← 增强参数
      └── 新增 crop_x, crop_y, crop_w, crop_h 参数
      └── 新增 remove_bg: boolean 参数
```

**推荐模型/库：**

- `@imgly/background-removal` — 浏览器端或 Node.js 端运行，基于 ONNX Runtime
- `sharp` — 已有依赖，用于裁剪和基本处理
- 智能裁剪可用简单的显著性检测算法（边缘密度、色彩对比度）

**预计工作量：** 2-3 周

- 后端去背景服务：3 天
- 图片分析接口：2 天
- 前端参数面板重构：5 天
- 并排预览对比：3 天
- 测试调优：3 天

### 方案二：AI 文字生成拼豆图案（优先级：P0）

**功能描述：**
用户在输入框中输入描述文字（如"一只可爱的柴犬，卡通风，简单线条"），AI 生成对应的像素画，然后自动匹配珠子颜色，用户可直接在编辑器中使用和修改。

**使用场景：**

- 用户不知道自己要什么图案，想尝试一些创意
- 用户需要特定主题的图案（节日、宠物、食物等）
- 用户想快速获得灵感

**技术实现：**

```
方案 2A（推荐起步）：调用第三方 API

前端：
  ├── TextToPatternView.vue（新增页面）
  │   ├── 文字输入框 + 风格选择器（卡通/写实/极简/像素风）
  │   ├── 网格尺寸选择
  │   └── 生成结果展示 → 一键导入编辑器
  │
后端：
  ├── POST /api/text-to-pattern    ← 新增
  │   ├── 1. 调用 Replicate API（Retro Diffusion 或 SD-πXL）
  │   ├── 2. 接收生成的像素画图片
  │   ├── 3. Sharp → 缩放至目标网格 → CIEDE2000 颜色匹配
  │   └── 4. 返回 grid_data 给前端
  │
  └── 或使用硅基流动/火山引擎等国内 API（响应更快，中文支持更好）

方案 2B（后期自建）：自托管模型
  ├── 使用 ComfyUI + SDXL + LoRA（如 nerijs/pixel-art-xl）
  ├── GPU 服务器（RTX 4090 或更高）
  └── 优点：无 API 费用、可定制风格
  └── 缺点：硬件成本高、维护复杂
```

**推荐 API/模型：**

| 提供商    | 模型                      | 特点                   | 成本       |
| --------- | ------------------------- | ---------------------- | ---------- |
| Replicate | `retro-diffusion/rd-fast` | 15 种风格，秒级生成    | ~$0.002/次 |
| Replicate | `retro-diffusion/rd-plus` | 高质量，场景/图标      | ~$0.005/次 |
| Replicate | `sdxl-pixel-art`          | 通用像素画生成         | ~$0.003/次 |
| 硅基流动  | Stable Diffusion 系列     | 国内访问快，中文提示词 | 按量计费   |
| 火山引擎  | 即梦 AI                   | 字节跳动，中文优化     | 按量计费   |

**预计工作量：** 3-4 周（API 方案）

- 后端 API 对接 + 后处理管道：5 天
- 前端文字输入页面：5 天
- 生成结果预览 + 导入编辑器：3 天
- 风格预设和参数调节：3 天
- 错误处理 + loading 状态：2 天
- 测试调优：3 天

### 方案三：AI 智能配色助手（优先级：P1）

**功能描述：**
为编辑器提供智能配色功能，帮助用户更高效地完成设计：

1. **调色板推荐** — AI 根据已有图案内容推荐协调的颜色组合
2. **颜色自动填充** — 用户选择区域 + 颜色主题 → AI 自动生成渐变/纹理填充
3. **色彩和谐度检测** — 分析设计中的颜色搭配，给出可读性和美观度评分
4. **色弱友好检查** — 检测配色是否对色弱用户友好

**技术实现：**

```
后端新增服务：
  ├── server/services/paletteAdvisor.js  ← 新增
  │   ├── paletteRecommend(grid_data)    — 分析现有颜色，推荐互补/相似色
  │   ├── harmonyScore(grid_data)        — 基于色彩理论打分
  │   └── colorblindCheck(grid_data)     — 模拟色弱视角检查对比度
  │
  └── 算法基础：
      ├── 色轮理论（互补色、三角色、邻近色）
      ├── WCAG 对比度计算
      └── 基于已选品牌珠子颜色的实际可用色

前端集成：
  ├── EditorView.vue 增强
  │   ├── 侧边栏新增「配色助手」面板
  │   ├── 点击颜色 → 显示推荐替代色（跨品牌映射已有）
  │   ├── 所选区域 → 「自动填充」按钮 → 调用算法生成填充
  │   └── 「配色检查」按钮 → 显示和谐度报告
```

**为什么不直接用 AI 大模型：**

- 颜色搭配有成熟的色彩理论（Itten 色轮、Kuler 算法）
- 规则引擎 + 色差计算更可控、更快、无 API 成本
- 可以在规则引擎基础上，后续接入 LLM 做「自然语言描述配色」（如"给我配一个海滩日落风格"）

**预计工作量：** 2-3 周

- 配色推荐算法：3 天
- 色彩和谐度评分：2 天
- 色弱友好检查：1 天
- 前端配色面板：5 天
- 编辑器集成（自动填充）：3 天
- 测试调优：2 天

### 方案四：AI 图纸识别增强（优先��：P1）

**功能描述：**
升级现有 OCR 图纸识别功能，提高识别准确率和用户体验：

1. **图像预处理增强** — 透视校正 + 自适应二值化 + 噪声去除
2. **网格线自动检测升级** — 从简单的亮度投影升级为边缘检测 + Hough 变换
3. **色号区域分割** — 基于颜色聚类的色号区域定位
4. **批量识别** — 支持一张照片中有多个图纸的批量识别
5. **置信度标注** — 对识别结果中的每个格子标注置信度，低置信度格子高亮提示用户手动修正

**技术实现：**

```
后端增强（server/services/ocr.js）：
  ├── 图像预处理管道：
  │   ├── sharp 透视校正（四点变换）
  │   ├── 自适应阈值二值化
  │   ├── 形态学操作（膨胀/腐蚀去除噪点）
  │   └── 边缘增强
  │
  ├── 网格检测升级：
  │   ├── Canny 边缘检测 → Hough 线检测
  │   ├── 线聚类分组 → 确定网格行列数
  │   └── 交点校验 + 异常修正
  │
  ├── 色号区域定位：
  │   ├── 颜色空间聚类（K-Means, k=背景+网格线+色号）
  │   ├── 连通域分析 → 定位每个色号文字区域
  │   └── 逐区域裁剪 → Tesseract.js OCR
  │
  └── 置信度计算：
      ├── OCR 置信度 + 颜色匹配置信度
      └── 低于阈值的格子标记 is_low_confidence: true

前端增强（OcrView.vue）：
  ├── 四角拖拽校正（用户手动辅助）
  ├── 低置信度格子红色闪烁高亮
  ├── 批量识别结果缩略图列表
  └── 逐格子点击修正（弹出颜色选择器）
```

**可选深度学习增强（Phase 2）：**

- 用 YOLOv8 训练色号区域检测模型 → ONNX 导出 → 浏览器推理
- 或用 Ultralytics 在服务端运行

**预计工作量：** 3-5 周

- 图像预处理管道：5 天
- Hough 网格检测：4 天
- K-Means 色号区域分割：3 天
- 置信度系统：2 天
- 前端交互增强（拖拽校正、高亮提示）：5 天
- 批量识别支持：2 天
- 测试 + 准确率优化：5 天

### 方案五：AI 风格迁移与图案增强（优先级：P2）

**功能描述：**
为设计图案提供 AI 增强处理：

1. **艺术风格迁移** — 将设计转成不同像素艺术风格（Minecraft 风、GBA 风、复古 8-bit 等）
2. **图案简化** — 自动减少颜色数，生成更简洁的版本（适合小尺寸拼豆板）
3. **细节增强** — AI 添加阴影、高光、轮廓等细节，让图案更立体
4. **无缝平铺** — 将图案转成可无缝拼接的连续图案（适合做背景/边框）

**技术实现：**

```
方案 5A（轻量版，规则算法）：
  ├── 图案简化：K-Means 颜色聚类 + 合并最小区域
  ├── 轮廓增强：边缘检测 + 加深相邻不同色珠子
  ├── 无缝平铺：镜像边缘 + 边界融合算法
  └── 优点：零依赖、即时响应、确定性输出
  └── 工作量：2-3 周

方案 5B（AI 增强版，需 GPU）：
  ├── 风格迁移：ControlNet (Canny) + LoRA 风格模型
  ├── 细节增强：针对拼豆优化的超分辨率模型
  ├── 需要 GPU 服务器或使用 Replicate API
  └── 优点：效果更好，风格多样
  └── 工作量：4-6 周
```

**推荐先做 5A（轻量版），积累用户反馈后再决定是否投入 5B。**

### 关键决策点

1. **文字生成功能（方案二）的 API 选择**：
   - 推荐先用 **硅基流动**（国内，中文提示词好，延迟低）

   - Phase 1 用传统 CV 算法（OpenCV 类似操作在 Node.js 中可用 `sharp` + 像素计算替代）
   - 如果传统算法准确率不足 80%，再引入 YOLOv8 色号检测

## 七、总结

豆丁（Douding）已经具备了扎实的基础能力——CIEDE2000 色差匹配、Floyd-Steinberg 抖动、Tesseract.js OCR、三层 Canvas 编辑器。在此基础上，AI 功能的引入应该分层次、分阶段进行：

**立即可做（无 API 依赖、纯算法）：**

- 智能配色助手（方案三）
- 图片转图纸增强 — 智能裁剪建议 + 参数推荐（方案一子集）

**短期可做（需要 API 但实现简单）：**

- AI 自动去背景（方案一完整版）
- 文字生成拼豆图案（方案二 API 版）

**中期可做（技术难度较高但差异化强）：**

- 图纸识别增强（方案四）
- 风格迁移轻量版（方案五）

这些方案在技术栈（Node.js + Sharp + Vue 3 Canvas）上完全兼容当前架构，不存在需要大规模重构的风险。建议按 Phase 1 → Phase 2 → Phase 3 的顺序逐步落地。
