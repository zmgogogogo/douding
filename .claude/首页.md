一、产品概述
1.1 核心价值与作用
首页是产品的「门面与枢纽」，承担四大核心职能：
功能导航枢纽：集中所有核心工具入口，用户可一键触达转图、创作、豆仓、辅助制作等核心功能，降低操作路径。
内容种草阵地：通过优质模板、大神作品、教程内容激发用户创作欲，提升用户使用时长与创作转化率。
运营活动触达：承载 Banner、弹窗、活动入口，是新品功能、运营活动、会员转化的核心触达渠道。
用户留存载体：通过每日更新的推荐内容、签到任务、个性化推荐提升用户回访频次与留存率。
1.2 核心设计原则
创作优先原则：核心创作入口突出展示，首屏即可触达最高频的「图片转拼豆」「空白创作」功能，减少用户决策成本。
分层信息原则：按「高频功能→运营活动→分类导航→内容流」从上到下排布，信息权重依次递减，符合用户浏览习惯。
千人千面原则：新老用户、不同偏好用户展示差异化内容，新用户重引导、老用户重推荐，会员用户展示专属权益。
性能极致原则：首屏加载＜1.5 秒，列表滑动流畅无卡顿，图片懒加载与缓存机制完善，弱网环境仍有基础体验。
运营灵活原则：所有运营位、功能入口、分类排序支持后台配置，无需发版即可调整首页布局与内容。

三、各模块详细设计
3.1 顶部导航栏模块
3.1.1 模块定位
页面顶部常驻导航，提供品牌展示、全局搜索、消息通知、个人入口，是页面的基础框架。
3.1.2 布局与元素
从左到右依次排列：
品牌 Logo
样式：产品 Logo 图标 + 产品名称文字，左对齐，左侧内边距 16dp
尺寸：图标 24dp，文字 16sp 加粗
交互：点击刷新首页并回到顶部
搜索入口（折叠式，可选展开）
样式：放大镜图标 +「搜索模板 / 作品 / 教程」占位文字，浅灰色圆角背景
交互：点击跳转至全局搜索页，支持搜索历史、热门搜索推荐
功能图标区（从左到右）
签到图标：右上角小红点标记未签到，点击跳转签到任务页
消息图标：右上角数字红点标记未读消息数，点击跳转消息中心
个人头像：用户登录后显示圆形头像，未登录显示默认头像；点击跳转个人中心
每个图标尺寸 24dp，间距 16dp，右侧内边距 16dp
3.1.3 交互逻辑
页面向下滑动时，导航栏背景由透明过渡为白色，提升文字可读性
页面向上滑动回到顶部时，背景恢复透明
未读消息数＞99 时显示「99+」
点击个人头像，未登录状态跳转登录页，已登录跳转个人中心
3.1.4 异常状态
未登录：头像显示默认图标，消息图标不显示未读数
消息推送权限未开启：消息图标正常显示，不做强制提示
3.2 全局搜索栏模块
3.2.1 模块定位
全局内容检索入口，支持模板、作品、教程、色号多维度搜索，降低用户找内容的成本。
3.2.2 搜索页完整功能
搜索框：顶部固定，支持输入关键词，右侧「取消」按钮
搜索历史：最近 10 条搜索记录，支持单条删除、一键清空
热门搜索：运营配置的热门关键词，按热度排序，点击直接搜索
搜索分类 Tab：全部、模板、作品、教程、色号，搜索结果按分类展示
搜索结果展示：
模板 / 作品：双列瀑布流卡片，显示缩略图、标题、使用量
教程：列表卡片，显示封面、标题、阅读量
色号：色块 + 色号 + 名称，点击可查看详情、加入豆仓
3.2.3 搜索逻辑
支持模糊匹配、拼音匹配、同义词匹配
搜索结果按相关度排序，加权因子：使用量、点赞量、发布时间、质量分
搜索联想：输入时实时下拉展示联想关键词
敏感词过滤：违规关键词拦截，提示搜索无结果
3.3 轮播 Banner 运营区
3.3.1 模块定位
核心运营位，用于新品功能上线、节日活动、会员推广、话题活动等重要内容曝光。
3.3.2 布局与样式
尺寸：宽度铺满屏幕（左右各 16dp 边距），高度 160dp，圆角 12dp
数量：最少 1 张，最多 8 张
指示器：底部居中圆形指示器，当前页高亮
轮播规则：自动轮播，间隔 3 秒，滑动后重置计时；最后一张滑向第一张无缝衔接
3.3.3 元素与跳转规则
每张 Banner 包含：背景图、标题文字、副标题、跳转链接。
支持跳转类型：
内部页面：模板详情、活动专题页、功能页面、教程详情
外部链接：H5 活动页
小程序跳转：其他小程序
无跳转：纯展示
3.3.4 后台配置能力
支持增删改 Banner，上传图片、填写标题、设置跳转链接
支持设置上线 / 下线时间
支持设置展示优先级、排序
支持设置可见用户范围：全部用户、新用户、会员用户、指定人群
支持查看 Banner 点击量、曝光量数据
3.3.5 异常状态
无配置 Banner：隐藏该区域，不占位
图片加载失败：显示默认占位图
单张 Banner：不显示指示器，不自动轮播
3.4 核心功能入口区（四大金刚）
3.4.1 模块定位
产品最高频的 4 个核心功能入口，首屏强曝光，是用户完成核心创作的主要路径。
3.4.2 布局与样式
布局：4 个入口等分横向排列，每个入口为「图标 + 文字」上下结构
图标尺寸：48dp，圆角背景，每个功能对应不同主题色
文字：13sp，居中对齐，图标下方 4dp 间距
整体内边距：左右 16dp，上下 12dp

3.7 内容 Feed 流模块（核心模块）
3.7.1 模块定位
首页核心内容区域，通过优质模板、作品、教程内容留住用户，激发创作欲，提升使用时长与转化率。采用双列瀑布流布局，兼顾信息密度与视觉体验。
3.7.2 布局规则
双列等宽瀑布流，左右间距 12dp，上下卡片间距 12dp，左右边距各 16dp
卡片宽度自适应屏幕，高度按图片比例自适应，避免图片拉伸变形
支持上下滑动加载更多，无限下拉
分类切换时，列表平滑过渡，保留当前浏览位置
3.7.3 卡片类型与详情
共 4 种卡片类型，统一卡片框架，差异化内容展示。
类型 1：模板卡片（占比最高）
卡片结构（从上到下）：
缩略图：圆角顶部，宽度铺满卡片，高度按比例自适应
角标（左上角）：「官方」「会员」「热门」「NEW」标签
标题：14sp，最多两行，超出省略
信息栏（底部）：
左侧：豆子尺寸（如 80×100）、颜色数（如 12 色）
右侧：使用量数字 + 使用图标
操作按钮（右下角悬浮）：「使用」小按钮，点击直接进入编辑器
数据字段：模板 ID、标题、缩略图 URL、宽、高、颜色数、使用量、是否官方、是否会员、作者 ID
点击跳转：点击卡片跳转模板详情页，可查看大图、色号清单、相关推荐，一键开始制作
类型 2：用户作品卡片
卡片结构：
成品图缩略图（主视觉）
作者信息栏：头像（小）+ 昵称，最多一行
作品标题：14sp，最多一行
互动数据：点赞数 + 点赞图标；评论数 + 评论图标
交互：
点击卡片跳转作品详情页
点击点赞图标直接点赞，再次点击取消
长按可保存图片、分享、举报
卡片上显示「同款模板」按钮，点击跳转对应模板
类型 3：教程卡片
卡片结构：
教程封面图
教程标题：加粗，最多两行
标签：「入门」「技巧」「避坑」等
阅读量、收藏量
跳转：点击跳转教程详情图文页

卡片结构：

3.7.4 内容排序与推荐规则
推荐 Tab 排序规则（个性化推荐）
核心算法：协同过滤 + 内容权重 + 用户行为加权
加权因子：
内容质量分（30%）：清晰度、完成度、审核评分
互动数据（30%）：使用量、点赞量、收藏量、分享量
用户偏好（25%）：用户历史浏览、收藏、使用的品类偏好
新鲜度（15%）：新发布内容加权，鼓励新内容
打散规则：同一作者、同一品类内容不连续出现，保证内容多样性
运营置顶：支持运营置顶指定内容，固定在第 1/3/5 位
热门 Tab 排序规则
按综合热度分排序，热度分 = 使用量 ×0.4 + 点赞量 ×0.3 + 收藏量 ×0.2 + 分享量 ×0.1
每日凌晨更新一次热门榜单
近 7 天数据权重最高，近 30 天数据衰减
最新 Tab 排序规则
按发布时间倒序排列，最新发布在前
审核通过后才进入列表
优质新内容额外加权曝光
3.7.5 加载与分页规则
首屏加载：默认加载 20 条数据，优先加载首屏可见区域图片

加载状态：
加载中：底部显示加载中动画 + 「加载中...」文字
无更多数据：底部显示「— 没有更多内容了 —」
加载失败：底部显示「加载失败，点击重试」
下拉刷新：下拉触发刷新，重置列表，重新加载第一页数据，刷新成功后轻提示

5.3 推荐算法规则
冷启动策略：新用户无行为数据时，推荐热门、高使用量、低难度的入门模板
个性化推荐：基于用户历史浏览、收藏、使用、点赞行为，推荐同类内容
探索机制：每次推荐插入 10% 的新品类内容，挖掘用户潜在兴趣
负反馈过滤：用户不感兴趣、举报、点「不喜欢」的内容减少推荐
运营干预：运营可设置置顶内容、加权内容、屏蔽内容
5.4 缓存策略
本地缓存：首页首屏数据缓存到本地，下次进入先展示缓存，再请求新数据，实现秒开
图片缓存：图片三级缓存，已加载图片无需重复下载
缓存有效期：首页数据缓存有效期 1 小时，超时自动刷新
离线缓存：离线状态下展示缓存的历史数据，提示网络异常
六、性能优化方案
6.1 首屏加载优化
骨架屏：加载过程中显示骨架屏，替代空白加载，提升感知速度
接口合并：首屏所有基础数据合并为一个接口，减少网络请求次数
本地缓存优先：先展示本地缓存数据，再异步请求新数据，实现秒开
资源预加载：预加载首屏可见区域图片，非首屏图片延迟加载
接口数据精简：只返回首屏必要字段，减少数据包大小
6.2 列表性能优化
虚拟列表：长列表采用虚拟滚动，只渲染可视区域卡片，滑动到 100 条以上依然流畅
组件复用：列表项组件复用，避免频繁创建销毁 DOM / 组件
图片懒加载：滚动到可视区域才加载图片，未加载时显示占位图
图片优化：
根据屏幕尺寸返回对应分辨率图片，避免加载大图
优先使用 WebP 格式，减少图片体积
缩略图压缩，保证加载速度
防抖节流：滚动事件节流，避免频繁触发计算；搜索联想防抖
6.3 交互流畅度优化
列表滑动时暂停图片加载与非必要计算，保证滑动帧率
点击反馈即时响应，网络请求异步执行，不阻塞 UI
动画使用硬件加速，避免卡顿掉帧
长列表图片回收机制，释放内存，避免 OOM

八、异常与边界处理
8.1 网络异常
无网络时：展示网络异常占位页，带「点击重试」按钮
弱网环境：延长超时时间，展示加载中提示，失败后提示重试
接口请求失败：Toast 提示「网络异常，请稍后重试」，保留原有缓存数据
8.2 空数据状态
分类下无内容：展示空状态插画 + 「暂无内容，去看看其他分类吧」提示
搜索无结果：展示「未找到相关内容，换个关键词试试」
新用户无收藏 / 浏览：空状态引导去发现内容
8.3 内容异常
图片加载失败：显示默认灰色占位图，带图片图标
内容违规被删除：点击后提示「该内容已不存在」，自动从列表移除
审核中内容：不进入首页推荐池，仅作者本人可见
8.4 边界场景
无更多内容：列表底部显示「没有更多内容了」，不再触发加载
内容数量不足一屏：正常排列，底部留白，不强制撑满
标题过长：最多两行，超出省略号截断
数字过大：点赞数、使用量超过 1 万显示「1.2w」格式
8.5 权限异常
会员内容点击：非会员弹出会员开通引导弹窗
未登录操作：弹出登录引导，登录后自动继续操作
