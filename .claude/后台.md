一、产品概述
1.1 核心价值
管理后台是拼豆平台的「控制中枢」，核心解决四大问题：
内容管控：统一管理官方模板、用户作品、教程活动，实现内容审核、上下架、推荐加权全流程可控，保障内容合规与质量。
运营提效：所有运营位、活动、弹窗、工具入口支持可视化配置，无需发版即可快速调整首页布局与运营策略，响应业务需求。
数据驱动：全维度数据统计与可视化看板，实时监控平台核心指标，为运营决策提供数据支撑。
系统可控：用户账号、权限、基础配置、AI 接口、版本更新统一管理，操作全留痕，保障系统安全稳定。
1.2 核心设计原则
权限分级原则：基于 RBAC 角色权限模型，细粒度控制菜单、按钮、数据权限，不同岗位人员仅可见职责范围内的功能与数据。
操作留痕原则：所有重要操作（删除、修改配置、封禁用户、审核驳回）全部记录操作日志，支持追溯与审计。
配置化原则：前端展示、运营规则、算法参数尽可能后台可配置，减少硬编码，降低迭代成本。
高效审核原则：统一审核工作台，支持批量操作、快捷键审核，提升内容审核效率。
数据可视化原则：核心指标图形化展示，支持多维度筛选、导出报表，降低数据获取成本。
容错防护原则：危险操作二次确认，支持操作撤销 / 恢复，避免误操作导致不可逆损失。

2.2.2 顶部全局栏
高度：60px，白色背景，底部 1px 分割线
左侧：产品 Logo + 菜单折叠按钮 + 面包屑导航（当前页面路径，支持点击返回上级）
右侧（从左到右）：
全局搜索框：支持搜索用户、模板、订单、反馈
消息通知图标：系统通知、审核提醒、举报提醒，红点未读提示

2.2.3 主内容区
内边距：左右各 24px，上下 20px
通用结构：页面标题 + 筛选区 + 操作按钮区 + 数据表格 / 卡片 + 分页
设计规范：所有列表页统一表格样式，表单页统一表单布局，弹窗统一尺寸与交互
2.2.4 Z 轴层级
从下到上：页面背景 → 内容区 → 顶部 / 左侧导航 → 弹窗 / 抽屉 → 消息提示 → 全屏加载遮罩
2.3 通用交互规范
列表页通用规则
顶部为筛选条件区，支持多条件组合筛选、重置
筛选区右侧为操作按钮区：新增、批量操作、导出
中间为数据表格，支持列排序、单行操作
底部为分页器，支持切换每页条数、跳转指定页
表单页通用规则
标签左对齐，输入框右对齐，必填项红星标记
底部固定：取消、确定按钮
表单校验实时提示，错误标红并显示错误原因
弹窗通用规则
居中弹出，遮罩层半透明黑色
顶部标题 + 关闭按钮，底部操作按钮
点击遮罩 / 关闭按钮 / ESC 键可关闭
反馈规范
操作成功：右上角绿色 Success 提示，2 秒自动消失
操作失败：红色 Error 提示，需手动关闭或自动消失
加载中：按钮 loading 状态，禁止重复提交
危险操作：弹出二次确认弹窗，需手动确认

六、安全与风控
6.1 账号安全

删除保护：重要数据软删除，保留恢复入口，30 天后才物理删除
6.3 内容安全
机器审核前置：所有用户上传内容先经过内容安全接口检测，违规自动拦截
人工审核兜底：疑似违规内容进入人工审核队列
违规用户分级处罚：警告、禁言、短期封禁、永久封禁
敏感词库定期更新，违规词自动替换 / 拦截
6.4 系统安全
接口防刷：基于用户 / IP 的限流策略
防 CSRF、防 XSS、防 SQL 注入
后台访问 IP 白名单（可选），限制指定 IP 才可访问
定期安全漏洞扫描，及时修复高危漏洞
